二． 将本地git仓库发布到GitHub上

详细步骤

1.命令行”cd ~/.ssh “如果没报错或者提示什么的说明就是以前生成过直接使用 cat ~/.ssh/id_rsa.pub命令就是可以查看本机上的 SSH key

2.如果没有生成过则需创建

  • a) git config --global user.name "用户名"

  • b) git config --global user.email "邮箱地址"

  • c)

  • d) 生成密钥 SSH key—-> ssh-keygen -t rsa -C '上面的邮箱'

  • e)

  • f) 将SSH key添加到Github中相关联——>设置——>SSH和GPG密匙 相关联即可，key在ssh下 id_rsa.pub中，复制粘贴到GitHub中设置

  • g) 检查是否成功–>ssh -T git@github.com若返回 hi…successfully即成功

  • h) #将本地仓库文件上传到Github

  • i)

  • j) ###将GitHub上的文件下拉到本地仓库

  • k) git pull

三． 总结：

  • 其实只需要进行下面几步就能把本地项目上传到Github

  • 1、在本地创建一个版本库（即文件夹），通过git init把它变成Git仓库；

  • 2、把项目复制到这个文件夹里面，再通过git add .把项目添加到仓库；

  • 3、再通过git commit -m "注释内容"把项目提交到仓库；

  • 4、在Github上设置好SSH密钥后，新建一个远程仓库，通过git remote add origin https://github.com/guyibang/TEST2.git将本地仓库和远程仓库进行关联；

  • 5、最后通过git push -u origin master把本地仓库的项目推送到远程仓库（也就是Github）上；（若新建远程仓库的时候自动创建了README文件会报错，解决办法看上面）。
