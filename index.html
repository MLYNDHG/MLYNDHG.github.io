Vue源码深入学习.第一篇

第一篇主要学习Vue.js内部核心技术“变化侦测”，以及从0到1简单实现一个“变化侦测”系统

[TOC]

一.Vue.js

1.1什么是Vue.js

定义：一个JavaScript框架。

渐进式：可直接嵌入Vue.js使用，可单页面组件式使用，可构建大型应用使用命令行快速构建项目工程。

二.Object的变化侦测

1.什么是变化侦测。

*****Object和Array的变化侦测采用不同的方式

定义：应用运行时内部状态会不断发生改变，那么就要不断的渲染，如何知道什么状态发生改变就需要进行侦测。

渲染：vue.js会根据状态生成dom输出到页面上显示查出来，这个过程叫做渲染。

变化侦测的类型：推|拉，Angular和React属于拉，vue属于推。

推：状态更新，会立马知道是具体哪一个更新了，颗粒度更细，依赖追踪耗费内存开销很大，vue2.0开始引入虚拟dom，加大侦测颗粒度，降低内存开销，推类型的变化侦测可以随意调整粒度。

拉：某个状态改变时，框架不知道具体哪一个改变，此时发送一个信号给框架，框架内部收到消息，进行暴力对比找到需要更新的dom节点进行重新渲染。

2.如何追踪变化

Vue3.0之前使用Defineproperty来追钟状态变化

Vue3.0使用Proxy

3.Object如何收集依赖

依赖：使用到当前状态的地方被称为依赖。

收集依赖：将使用到当前状态的地方全部收集起来，当状态改变时，循环触发一遍即可。（getter中收集依赖，setter中触发依赖）。

4.依赖（window.target）是谁?

状态发生改变后需要被通知修改的地方，可能是绑定的模板也可能是用户写的一个watch，多种情况统一处理，抽象为一个类，这里称为watcher，所以我们收集的是watcher！。

那么以后只要在状态改变，set触发通知watcher，然后watcher再通知分发给每一个依赖。

5.依赖怎么知道自己是谁的依赖，并被加入进dep中进行管理的？

解：上面说了依赖是通过getter来收集的，即获取该状态会触发收集依赖函数的getter方法，那么window.target只需要触发一下getter就可以主动把this添加到对应的dep中。

7.总结

变化侦测：侦测数据的变化，能发现变化并触发通知。

definedProperty：Object 使用此属性进行侦测。

dep：存储依赖。

getter：数据读取时会触发。

setter：数据修改时会触发。

依赖：watcher，当watcher触发getter时就会自动收集此依赖，哪个watcher触发了getter，就把哪个收集进入dep中。

通知：当状态发生改变，循环触发dep，将所有存储在dep中的watcher全部都通知一遍。

watcher原理：将自己设置为全局同一位置（设为window.target）,然后读取数据，读取数据就会触发这个数据的getter（即definedProperty的getter），那么definedProperty就会将此watcher收集到dep中。

Observer：将一个object中的所有数据包括子数据全部切换为响应式。

Redis

  • 下载安装

  1. JDK版本：至少使用JDK8版本

  2. 什么是Redis：

    基于内存亦可持久化的日志型 Key-Value数据库 Redis就像是一个HashMap 不在JVM中运行以独立进程的形式运行 当作缓存使用 比数据库(mysql)快，常用的数据放在这里可以提高性能

  3. 下载：http://redis.io windows版本的下载地址是： http://redis.io/download

  • 使用

  1. 服务器开启

    redis-server.exe

  2. 客户端访问

    redis-cli.exe

  • 常见命令
  1. ​ String（字符串）：set get incr type ttl rename
  2. ​ List（列表）：lpush rpush llen lrange lpop rpop
  3. ​ Hash（字典）：hset hgetall hkeys hvals
  4. ​ Set（集合）:SADD SMEMBERS
  5. ​ Sorted Set（有序集合）:zadd zcard zcount
  6. ​ 命令手册http://www.redis.cn/commands.html
  • Jedis
  1. 是什么：在java项目代码中使用redis，不仅仅是redis给的客户端中，只需引入jar包即可,即jedis.jar
  2. 使用：javaProject中可直接当作客户端来操作redis，但是redis服务端还是要开启（不然呢？？？）

  1. 开启redis服务端

数据结构第一章<绪论>

1.数据结构是什么

1.1.1数据结构的定义

几个概念术语先记住：

  • 数据：是描述客观事物的数和字符的集合

  • 数据元素：数据的基本单位

  • 数据项：具有独立含义的数据最小单位，也成为字段或者域

  • 数据对象：性质相同数据元素的集合，是数据的一个子集

  • 数据结构：数据元素与数据元素之间的关系，堪称相互之间存在特定关系的元素的集合

  • 逻辑结构：数据元素间的逻辑关系

  • 存储结构：数据元素及关系在计算机存储器中的存储表示

  • 运算:对数据的操作

定义：数据结构=逻辑结构+存储结构+运算

1.1.2逻辑结构
  • 是什么：具体问题抽象出来的逻辑关系
  • 有哪些：
    • 集合：除了在一个集合中没有其他关系
    • 线性：一对一，一个前驱，一个后继
    • 树型：一对多，一个前驱，多个后继
    • 图型：多对多，多个前驱，多个后继
1.1.3存储结构
  • 是什么：数据元素及关系在计算机存储器中的存储表示
  • 有哪些：
    • 顺序结构：
    • 链式结构：
    • 索引结构：存储数据元素信息时附加建立一个索引表
    • 散列结构：通过哈希值计算出元素地址
1.1.4数据运算
  • 是什么：对数据元素实施的操作
  • 使用：一种数据结构它的逻辑结构是单一的，但是存储结构需根据实际来选择，所以运算的定义是确定的，但是运算的实现需根据实际的存储结构来确定
1.1.5数据类型与抽象数据类型

概念：

  • 数据结构：一组性质相同值的集合与定义在此集合上一组操作的总成，是某种编程语言已经实现的数据结构
  • 抽象数据类型：在不考虑存储结构和运算具体实现算法前提下，从具体问题中抽象出来的逻辑数据结构与逻辑数据结构上的运算
1.2.1算法及其描述

概念：

  • 算法：对特定问题求解步骤的描述

  • 特性：

    • 有穷性
    • 确定性
    • 可行性
    • 有输入：零个或多个输入
    • 有输出：一个多个输出

  • 目标：

    • 正确性
    • 可使用性
    • 可读性
    • 健壮性
    • 高效率与底存储量需求
1.3.1算法分析

分析算法占用计算机资源的多少
