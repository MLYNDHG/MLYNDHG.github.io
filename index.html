数据结构第一章<绪论>

1.数据结构是什么

1.1.1数据结构的定义

几个概念术语先记住：

  • 数据：是描述客观事物的数和字符的集合

  • 数据元素：数据的基本单位

  • 数据项：具有独立含义的数据最小单位，也成为字段或者域

  • 数据对象：性质相同数据元素的集合，是数据的一个子集

  • 数据结构：数据元素与数据元素之间的关系，堪称相互之间存在特定关系的元素的集合

  • 逻辑结构：数据元素间的逻辑关系

  • 存储结构：数据元素及关系在计算机存储器中的存储表示

  • 运算:对数据的操作

定义：数据结构=逻辑结构+存储结构+运算

1.1.2逻辑结构
  • 是什么：具体问题抽象出来的逻辑关系
  • 有哪些：
    • 集合：除了在一个集合中没有其他关系
    • 线性：一对一，一个前驱，一个后继
    • 树型：一对多，一个前驱，多个后继
    • 图型：多对多，多个前驱，多个后继
1.1.3存储结构
  • 是什么：数据元素及关系在计算机存储器中的存储表示
  • 有哪些：
    • 顺序结构：
    • 链式结构：
    • 索引结构：存储数据元素信息时附加建立一个索引表
    • 散列结构：通过哈希值计算出元素地址
1.1.4数据运算
  • 是什么：对数据元素实施的操作
  • 使用：一种数据结构它的逻辑结构是单一的，但是存储结构需根据实际来选择，所以运算的定义是确定的，但是运算的实现需根据实际的存储结构来确定
1.1.5数据类型与抽象数据类型

概念：

  • 数据结构：一组性质相同值的集合与定义在此集合上一组操作的总成，是某种编程语言已经实现的数据结构
  • 抽象数据类型：在不考虑存储结构和运算具体实现算法前提下，从具体问题中抽象出来的逻辑数据结构与逻辑数据结构上的运算
1.2.1算法及其描述

概念：

  • 算法：对特定问题求解步骤的描述

  • 特性：

    • 有穷性
    • 确定性
    • 可行性
    • 有输入：零个或多个输入
    • 有输出：一个多个输出

  • 目标：

    • 正确性
    • 可使用性
    • 可读性
    • 健壮性
    • 高效率与底存储量需求
1.3.1算法分析

分析算法占用计算机资源的多少

  • 时间复杂度:原操作执行次数（频度）T(n),求出最高阶

  • 其实只需要进行下面几步就能把本地项目上传到Github

  • 1、在本地创建一个版本库（即文件夹），通过git init把它变成Git仓库；

HEXO使用文档

Hexo是什么：hexo是个人博客框架

Hexo怎么用：

1.安装Node.js和Git并注册GitHub

2.检验是否安装成功

  • git –version
  • node –version
  • npm –version

3.安装Hexo

  • npm install -g hexo-cli # -g 代表着全局安装

4.创建个人博客

  • $–>在hexo文件目录使用cmd命令行
  • $ hexo init blogName
  • $ cd blogName
  • $ npm install
  • 会得到一个博客文件夹

5.创建一篇文章

  • $ hexo new ‘文章标题’ 创建到/source/_posts文件夹下一个md文件，使用markdown语法编辑即可

6.编译+启动

  • $ hexo generate # 生成静态文件，可简写为 hexo g
  • $ hexo server # 启动服务器。默认情况下，访问网址为：http://localhost:4000/。可以简写为 hexo s

7.部署到 Github站点

  • 需要_config.yml 配置文件配置参数
  • 要安装一个部署插件 hexo-deployer-git。
  • npm install hexo-deployer-git –save

8.上传部署

  • hexo g -d
